# Writing Assistant 프로젝트 요약

## 1. 프로젝트 목적

Writing Assistant는 사용자가 작성 중인 글을 워드, 메모장, 브라우저 등 실제 작성 환경 위에서 바로 분석하고 보조하는 AI 글쓰기 도우미입니다.

핵심 목표는 사용자가 별도 창으로 문장을 옮기지 않아도, 현재 작성 중인 문맥을 기반으로 맞춤법 교정, 문체 변환, 요약, 제목 추천, 평가, 기록 관리를 수행하는 것입니다.

## 2. 주요 기능

### 텍스트 인식

- 클립보드 모드: 복사된 텍스트를 인식합니다.
- 드래그 모드: 사용자가 선택한 영역만 분석합니다.
- 실시간 모드: 커서 위치 기준 현재 문장, 현재 문단, 글 전체 범위를 선택해 분석합니다.
- 지원 환경: Word, Windows 메모장, 브라우저 확장 기반 웹 입력창.

### AI 기능

- 맞춤법 교정: 맞춤법, 띄어쓰기, 문법, 문장부호, 어색한 표현, 문맥상 맞지 않는 단어를 교정합니다.
- 비수정 맞춤법 안내: 직접 수정하지 않고 오류 위치에 UI 마커와 설명 오버레이를 표시합니다.
- 문체/말투 변환: 사용자가 선택한 말투 또는 즐겨찾기 말투로 문장을 변환합니다.
- 요약: 짧게, 핵심 bullet, 자세히 세 가지 방식으로 요약합니다.
- 제목 추천: 전체 글 제목 또는 선택 영역의 소제목을 추천하고 삽입합니다.
- 평가: 글의 점수와 평가 이유를 표시합니다.

### 오버레이 UI

- 메인 오버레이: 평가, 제목, 맞춤법, 문체, 요약 기능을 한 곳에서 실행합니다.
- 마커 오버레이: 맞춤법/문맥 오류 위치에 빨간 선과 유형 라벨을 표시합니다.
- 설명 오버레이: 오류 원문, 수정안, 이유, 교체 버튼을 제공합니다.
- 수정 이유 오버레이: AI가 수정한 이유를 스크롤 가능한 큰 창으로 보여줍니다.
- 알림 오버레이: AI 처리 상태, 기능 수행 정보, 오류 정보를 로컬 알림으로 확인합니다.

## 3. AI 처리 구조

클라이언트는 `AIClient`를 통해 로컬 FastAPI 서버에 요청합니다.

서버의 `AIService`는 OpenAI Responses API를 사용하며, 기능별로 다른 프롬프트와 JSON 스키마를 적용합니다.

맞춤법 교정 응답 구조:

- `corrected_text`: 전체 교정문
- `feedback`: 전체 교정 의견
- `corrections`: 오류별 원문, 수정안, 유형, 이유, 확신도

이 구조를 이용해 수정 모드에서는 전체 교정문을 적용하고, 비수정 모드에서는 각 오류별 마커와 설명 오버레이를 생성합니다.

## 4. DB 구조

기록 DB는 PostgreSQL을 기준으로 동작하며, 기존 단일 로그 테이블 중심 구조에서 요청 원문과 기능별 결과를 분리하는 구조로 개편했습니다.

### DB 엔진

- PostgreSQL
- SQLAlchemy ORM
- `DATABASE_URL=postgresql+psycopg2://...` 환경변수 기반 연결

### 기준 테이블

- `analysis_requests`
  - 사용자가 분석을 요청한 원문을 저장합니다.
  - 문서별 기록 묶음의 기준이 됩니다.

### 기능별 결과 테이블

- `spelling_results`: 맞춤법 교정 결과와 교정 이유 저장
- `summary_results`: 요약 결과 저장
- `tone_results`: 문체/말투 변환 결과 저장
- `evaluation_results`: 평가 점수와 이유 저장
- `title_results`: 제목 추천 결과 저장

이 구조를 통해 하나의 원문에 여러 기능 결과를 연결할 수 있고, 기록 상세에서 아직 호출하지 않은 기능을 나중에 추가로 호출할 수 있는 기반을 마련했습니다.

## 5. 클라이언트 구조

### 주요 UI 파일

- `client/ui/main_window.py`: 앱의 중심 제어, 입력 이벤트, AI 기능 호출, 기록 저장
- `client/ui/main_overlay.py`: 외부 프로그램 위에 표시되는 메인 오버레이
- `client/ui/spelling_inspection_overlay.py`: 비수정 맞춤법 마커와 설명 오버레이
- `client/ui/mini_overlay.py`: 문체 선택, 즐겨찾기, 보조 오버레이
- `client/ui/result_panel.py`: 메인 프로그램 창, 설정, 기록, 결과 탭

### 주요 입력/적용 파일

- `client/input/ai_grammary_text_reader.py`: Word/메모장 등 외부 창 텍스트 인식
- `client/input/output_applier.py`: 교정 결과를 원본 위치에 적용하고 Word 서식을 보존
- `browser_extension/content.js`: 웹 입력창 텍스트 인식 및 교체

## 6. 브라우저 확장 개선

브라우저 확장은 사용자가 실제로 클릭하거나 입력한 편집 영역을 우선 추적합니다.

개선 포인트:

- Gmail, Naver Cafe, SmartEditor 등 편집기 후보를 별도 점수화
- 최근 상호작용한 입력창 우선 선택
- `textarea`, `input`, `contenteditable`, `role=textbox` 지원
- 팝업 UI를 통해 현재 입력창 텍스트 확인 및 적용 가능

## 7. 서식 보존

Word 교정 적용 시 원문 전체 서식을 하나로 덮지 않고, 문자 인덱스 단위로 원문 서식을 교정문에 매핑합니다.

교정문이 길어질 경우에는 원문 마지막 글자의 서식을 연장하고, 짧아질 경우에는 사라진 글자의 서식을 적용하지 않습니다.

이를 통해 빨간 글자, 굵게, 형광펜, 글자 크기 등 기존 Word 서식이 최대한 보존됩니다.

## 8. 기록 및 로그인

- 로그인 사용자는 기록 저장을 켜고 끌 수 있습니다.
- 기록은 현재 문서명 또는 원문 기준으로 묶어 볼 수 있습니다.
- 기록 상세에서 원문, 결과, 점수, 이유, 문체, 제목 정보를 확인할 수 있습니다.
- 기록 삭제 기능을 제공합니다.
- 비로그인 상태에서 기록 버튼을 누르면 로그인 필요 화면으로 이동합니다.

## 9. 배포 구조

PyInstaller `onedir` 방식으로 exe를 생성합니다.

빌드 산출물:

- `dist/WritingAssistant/WritingAssistant.exe`

exe 환경에서는 로컬 FastAPI 서버를 별도 Python 프로세스가 아니라 앱 내부 백그라운드 스레드로 실행하도록 처리했습니다.

## 10. 시연 포인트

1. Word에서 드래그 모드로 일부 문장만 맞춤법 검사
2. 비수정 모드에서 오류 마커와 설명 오버레이 표시
3. 교체 버튼으로 특정 오류만 수정
4. 수정 모드에서 전체 교정 적용 및 수정 이유 확인
5. 문체 프리셋/즐겨찾기로 문장 말투 변경
6. 요약 방식별 결과 차이 확인
7. 제목/소제목 추천 및 삽입
8. 기록 화면에서 같은 원문 기준 기능 결과 묶음 확인
9. 브라우저 입력창에서 확장 기능으로 텍스트 인식

## 11. 프로젝트 의의

Writing Assistant는 단순한 텍스트 입력형 AI 도구가 아니라 실제 작성 환경에 직접 붙는 보조 시스템입니다.

외부 프로그램 인식, 오버레이 UI, AI JSON 응답 구조화, 원본 위치 교체, Word 서식 보존, 기록 DB 구조화가 결합되어 실제 사용 흐름에 가까운 글쓰기 보조 경험을 제공합니다.
